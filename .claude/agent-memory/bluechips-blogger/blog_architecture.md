## Blog Framework
- **Framework:** Astro (static output)
- **Location:** `blog/` directory in the monorepo root (E:\projects\BluechipsLondon\blog\)
- **Content files:** `blog/src/content/blog/*.md` — plain Markdown with YAML frontmatter
- **Config:** `blog/astro.config.mjs` — site URL is `https://blog.bluechips.live`

## Frontmatter Schema (from `blog/src/content/config.ts`)
All fields required unless marked optional:
```yaml
title: string            # 50–60 chars ideal for SERP
description: string      # 140–160 chars for SEO meta
publishDate: date        # YYYY-MM-DD
updateDate: date         # optional
author: string           # default "Bluechips London Editorial"
tags: string[]           # lowercase, hyphenated
category: string         # optional — "Borough Guides", "Practical Guides", "Safety & Trust", "Verification", "Services Explained", "Legal & Compliance"
keywords: string[]       # SEO keywords array
draft: boolean           # default false — set true to hide from listing
```

## Routing Convention
- Slug = filename without `.md` extension
- URL = `https://blog.bluechips.live/{slug}/`
- No manual manifest needed — Astro auto-discovers all non-draft files in `blog/src/content/blog/`

## Sitemap
- Auto-generated by `@astrojs/sitemap` at build time
- Outputs to `blog/dist/sitemap-index.xml` and `blog/dist/sitemap-0.xml`
- No manual update required — just add files and build

## RSS Feed
- Auto-generated at `blog/dist/rss.xml` from `blog/src/pages/rss.xml.ts`
- Picks up all non-draft posts automatically

## Deployment
- Vercel project root directory = `blog/` (critical setting)
- Auto-deploys on every push to `main` branch
- Build command: `npm run build` (runs `astro check && astro build`)
- Build output: `dist/`
- To add posts: create `.md` file in `blog/src/content/blog/`, commit, push — Vercel handles the rest

## Local Build/Dev
```bash
cd blog
npm run dev      # dev server at http://localhost:4321
npm run build    # production build to dist/
npm run preview  # preview built site
```

## Internal Linking Conventions
- Links to other blog posts use relative paths: `/some-post-slug`
- Links to the main site use absolute URLs: `https://bluechips.live/...`
- Key target pages on the main site:
  - `https://bluechips.live/escorts` — main directory
  - `https://bluechips.live/escorts?blue_tick_only=true` — verified only filter
  - `https://bluechips.live/escorts?borough_slug={slug}` — borough filter
  - `https://bluechips.live/join` — companion join/register
  - `https://bluechips.live/dashboard/verify` — companion verification portal

## SEO Components
- Article schema JSON-LD generated automatically in `BlogPostLayout.astro`
- BreadcrumbList schema generated automatically in `BlogPostLayout.astro`
- Blog schema on index page in `index.astro`
- OpenGraph + Twitter Card meta tags via `BaseLayout.astro`
- Canonical URLs auto-built from `Astro.site` + slug
- No react-helmet-async — this is Astro, not React
